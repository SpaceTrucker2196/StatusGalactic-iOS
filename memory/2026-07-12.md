### ⚠️ REAL FINDING — RepeaterBook API now requires auth (affects iOS too)

RepeaterBook `export.php` returns `HTTP 401 {"ok":false,"error_code":"auth_missing","message":
"Authorization required."}` for ALL user-agents (tested app UA, iOS-style UA, descriptive UA).
The endpoint is **no longer free/no-auth** — this postdates the iOS `RepeaterBookClient` which
still assumes "Free, no auth, just User-Agent required" and sends no token. So **A18 RepeaterBook
is non-functional on BOTH platforms** until a RepeaterBook API key is added (register + pass
Authorization). This is a product decision (add a RepeaterBook key to Settings like aprs.fi/nasa,
or drop the feature) that ALSO needs an iOS-side fix. Left the Android code as-is — it correctly
degrades to the 401 error footer. **Flagged for Jeff; not silently patched.**
