# 2026-05-19 — Home-screen widget (M14)

## Scope

Milestone M14 of `ROADMAP.md`: a WidgetKit extension that renders current brief headlines in small and medium home-screen sizes. No backend change.

## Files added

- `StatusGalacticWidget/StatusGalacticWidgetBundle.swift` — `@main WidgetBundle` exposing one widget.
- `StatusGalacticWidget/BriefWidget.swift` — `Widget` with `StaticConfiguration`. Supported families: small + medium.
- `StatusGalacticWidget/BriefWidgetProvider.swift` — `TimelineProvider`. `getTimeline` fetches one brief, returns one entry, with `.after(now + 30 min)` reload policy.
- `StatusGalacticWidget/BriefWidgetView.swift` — small and medium view variants:
  - **Small:** location, current temperature, current condition, next sun event with relative-time label.
  - **Medium:** + sunrise / sunset / next golden hour + moon phase + Kp index with status word.
- `StatusGalacticWidget/SharedConfig.swift` — `WidgetConfig` with hardcoded fallbacks for server URL and default lat/lng. Documents the App Group migration path.
- `StatusGalacticWidget/Info.plist` — `NSExtensionPointIdentifier: com.apple.widgetkit-extension`, `NSAllowsLocalNetworking: true` for dev.

## Design decisions

1. **Direct compilation, no shared framework.** WidgetKit extensions are tiny and compile fast; pulling a single model file and one API client into both targets is simpler than maintaining a third target. Easy to refactor into a framework if the widget grows.
2. **Static configuration, not intent.** The first widget has no user-selectable parameters because we have no App Group yet. Intent-driven configuration (callsign picker, marine zone, server URL) is a follow-up once App Groups are wired up.
3. **30-min reload hint.** Brief data is dominated by hourly-stable inputs (NWS forecasts, sun events). 30 min hint keeps the widget feeling alive while leaving room for OS throttling.
4. **Hardcoded fallbacks in `WidgetConfig`.** Without App Groups the widget cannot read `UserDefaults.standard` written by the app. Documented migration: add the `group.io.river.statusgalactic` entitlement to both targets and swap to `UserDefaults(suiteName:)`. Requires a DEVELOPMENT_TEAM.
5. **Next-sun-event detection.** Helper picks the first chronologically-future event from `[golden hour evening, sunset, astro dusk, sunrise]` so the small widget always shows whatever's about to happen.

## Drift / parity risks for Android

1. **Widget surface mismatch.** Android's `Glance` API is similar but not identical (uses `GlanceModifier` instead of SwiftUI modifiers, `androidx.glance.appwidget`). The data model and reload cadence can be 1:1; the view code cannot.
2. **App Group equivalent.** Android uses `DataStore` or shared `SharedPreferences` between app process and widget. Document the move from `WidgetConfig` hardcodes to shared store identically across platforms when that lands.
3. **Reload policy.** WidgetKit accepts a hint; the OS throttles. Glance uses `actionRunCallback` and `WorkManager` for scheduled updates. Periodicity should match (~30 min) but mechanism differs.
4. **Moon-phase glyph mapping.** iOS uses SF Symbols `moonphase.*`; Android needs a vector drawable set. Keep the same eight phase names so the lookup table maps identically.
5. **Next-sun-event helper.** Pure data logic, port verbatim. Encode the priority ordering (golden hour → sunset → astro dusk → sunrise) in a shared spec table in `MIGRATION_PLAN.md`.

## App Group migration path (when DEVELOPMENT_TEAM is set)

1. In Apple Developer, create App Group `group.io.river.statusgalactic`.
2. Add entitlements to both targets in `project.yml`:
   ```yaml
   settings:
     base:
       CODE_SIGN_ENTITLEMENTS: StatusGalactic/StatusGalactic.entitlements
   ```
3. Replace `WidgetConfig.defaultServerURL` and `defaultLatitude/Longitude` reads with:
   ```swift
   let shared = UserDefaults(suiteName: "group.io.river.statusgalactic")!
   ```
4. Update `ServerConfig` and `BriefView` in the main app to mirror their reads/writes to the same suite.
5. Drop `WidgetConfig`. Document the schema (keys, types) in this audit so the Android `DataStore` mirrors it.
